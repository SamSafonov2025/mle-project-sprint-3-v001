# Мониторинг

Дашборд предназначен для контроля FastAPI-сервиса, который онлайн оценивает
стоимость объектов недвижимости. Он должен быстро отвечать на вопросы:
работает ли сервис, есть ли ошибки, растет ли latency, поступает ли нагрузка и
какие значения предсказаний возвращает модель.

## Для мониторинга выбраны метрики нескольких слоев

### Инфраструктурный слой

- `up{job="ml-service"}` - доступность target в Prometheus.

Эта метрика показывает, может ли Prometheus опрашивать микросервис. Для нее
подходит stat/gauge-панель, потому что важен текущий статус: 1 означает, что
target доступен, 0 - что сервис или сеть недоступны.

### Метрики реального времени

- `rate(http_requests_total[1m])` - интенсивность HTTP-запросов.
- `histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))`
  - p95 latency базового FastAPI exporter.

Эти метрики помогают увидеть текущую нагрузку и деградацию времени ответа.
Для них используются time series графики: важна динамика во времени, особенно
после запуска скрипта симуляции нагрузки.

### Метрики прикладного уровня

- `ml_service_predictions_total{status="success"}` и
  `ml_service_predictions_total{status="error"}` - число успешных и ошибочных
  предсказаний.
- `ml_service_prediction_latency_seconds_bucket` - latency именно участка
  инференса модели.
- `ml_service_last_prediction_rub` - последнее предсказанное значение цены.
- `ml_service_requested_total_area_sqm_bucket` - распределение площади объектов
  в запросах.

Эти метрики выбраны, потому что для ML-сервиса недостаточно знать только HTTP
статус. Нужно понимать, что модель реально вызывается, как быстро она отвечает,
не растет ли доля ошибок и не приходит ли поток запросов с нетипичными
значениями признаков. Для счетчиков используется stat/table, для latency и
распределения площади - time series или histogram, для последнего prediction -
gauge/stat.

## Панели дашборда

Дашборд `dashboard.json` содержит минимум четыре аналитические панели:

1. Service availability - текущий `up`.
2. Request rate - скорость HTTP-запросов.
3. Prediction status totals - успешные и ошибочные предсказания.
4. Prediction latency p95 - задержка инференса.
5. Last predicted price - последнее значение прогноза.
6. Requested total area distribution - распределение площадей в запросах.

Используются разные виды визуализаций: stat, time series, gauge/histogram или
table. Такой набор покрывает инфраструктуру, приложение и ML-специфичные
сигналы.
